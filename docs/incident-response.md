## 重大度の定義

| 重大度 | 定義 | 例 |
|---|---|---|
| **P0（緊急）** | サービス全停止 / 個人情報漏えいの疑い | APIキー流出・サービスダウン |
| **P1（高）** | 主要機能が利用不可 / 不正利用の疑い | AI返答が出ない・大量リクエスト |
| **P2（中）** | 一部機能の不具合 | UI崩れ・表示エラー |
| **P3（低）** | 軽微な不具合・改善要望 | 文言ミス・デザイン |

---

## 初動フロー（P0・P1）

```
1. 検知
   └─ ユーザーからの報告 or 自分で気づく
      ↓
2. 状況確認（15分以内）
   ├─ Streamlit Cloud のダッシュボードでアプリ状態を確認
   ├─ Anthropic コンソールで API の状態・使用量を確認
   └─ GitHub リポジトリに不審なコミットがないか確認
      ↓
3. 切り分けと対応
   ├─ [APIキー漏えい疑い] → Anthropic コンソールでキーを即時無効化 → 新しいキーを発行 → Streamlit Secrets を更新
   ├─ [サービスダウン]    → Streamlit Cloud を再起動 → 改善しなければ GitHub の直前コミットを revert
   └─ [不正大量利用]      → Anthropic コンソールで使用量を確認 → 必要に応じてサービスを一時停止
      ↓
4. 関係者への報告
   ├─ 社内：口頭または Slack で報告（即時）
   └─ [個人情報漏えいの可能性がある場合]
      ├─ 影響を受けるユーザーに通知（メール等）
      └─ 個人情報保護委員会への報告義務を確認（漏えい発覚から72時間以内）
      ↓
5. 復旧確認・再発防止
   └─ 復旧後、原因と対応策をこのドキュメントに追記
```
